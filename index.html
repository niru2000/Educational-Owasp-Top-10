OWASP Top 10 Vulnerabilities

1. Injection

Injection flaws occur when untrusted data is sent to an interpreter as part of a command or query. The attacker's input can trick the interpreter into executing unintended commands or accessing data without proper authorization.

Example: Consider a simple login form that is vulnerable to SQL injection:


SELECT * FROM users WHERE username = 'admin' -- ' AND password = 'password';

2. Broken Authentication

Broken authentication occurs when application functions related to authentication and session management are implemented incorrectly, allowing attackers to compromise passwords, keys, or session tokens, or to exploit other implementation flaws to assume other users’ identities.

Example: An application might allow weak passwords, making it easier for attackers to guess user credentials using brute force.

3. Sensitive Data Exposure

Sensitive data exposure happens when applications do not adequately protect sensitive information, such as financial, healthcare, or personal data. Attackers may steal or modify such data to conduct fraud or other crimes.

Example: An application transmits sensitive data, like credit card numbers, over an unencrypted connection (HTTP instead of HTTPS).

4. XML External Entities (XXE)

XXE attacks occur when an XML processor parses XML input containing a reference to an external entity. This reference can be exploited to expose internal files, make HTTP requests, and more.

Example: An XML document with an external entity reference:

<!DOCTYPE foo [<!ELEMENT foo ANY > <!ENTITY xxe SYSTEM "file:///etc/passwd" >]>
<foo>&xxe;</foo>

5. Broken Access Control

Broken access control occurs when restrictions on authenticated users are not properly enforced. Attackers can exploit these flaws to access unauthorized functionality or data, such as accessing another user’s account.

Example: An attacker might modify the URL from /user/123/profile to /user/124/profile to access another user's profile without authorization.

6. Security Misconfiguration

Security misconfiguration occurs when security settings are defined, implemented, and maintained inappropriately. This vulnerability can result from failing to secure the application, web server, database, or other components.

Example: Leaving default configurations (such as default usernames and passwords) unchanged, allowing attackers to gain access.

7. Cross-Site Scripting (XSS)

XSS occurs when an application includes untrusted data in a web page without proper validation or escaping. Attackers can execute scripts in the user's browser, which can lead to session hijacking, defacement, or redirecting users to malicious sites.

Example: A script that executes within the victim’s browser:

<script>alert('XSS')</script>

8. Insecure Deserialization

Insecure deserialization occurs when untrusted data is used to abuse the logic of an application, leading to attacks such as remote code execution, replay attacks, and privilege escalation.

Example: An attacker manipulates serialized objects, leading to unauthorized actions when these objects are deserialized.

9. Using Components with Known Vulnerabilities

This vulnerability occurs when applications use libraries, frameworks, or other software components with known vulnerabilities. Attackers can exploit these vulnerabilities to take control of the server or application.

Example: An application uses an outdated version of a framework that has a known vulnerability, making it easier for attackers to exploit.

10. Insufficient Logging & Monitoring

Insufficient logging and monitoring allow attackers to further attack systems, maintain persistence, pivot to more systems, and tamper with, extract, or destroy data. Many breaches involve undetected incidents that remain unnoticed for extended periods.

Example: An application fails to log critical security events, such as multiple failed login attempts, making it difficult to detect brute-force attacks.
